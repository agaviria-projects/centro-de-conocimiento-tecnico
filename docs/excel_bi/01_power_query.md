> 📚 **Curso:** Excel BI para Analistas de Datos  
> 📖 **Módulo:** 01 - Power Query  
> 🎯 **Nivel:** Básico  
> ⏱️ **Duración estimada:** 15 minutos

---

# 🎯 Objetivo

Comprender qué es Power Query, para qué fue creado y cuál es su función dentro de un proyecto de análisis de datos.

Al finalizar este módulo entenderás por qué Power Query siempre debe ser el primer paso antes de construir un Modelo de Datos o utilizar Power Pivot.

---

# 🤔 Antes de Power Query...

Imagina que cada mañana recibes un archivo Excel con la información de la operación.

Todos los días debes realizar exactamente las mismas tareas:

- Eliminar columnas.
- Cambiar nombres.
- Convertir fechas.
- Corregir tipos de datos.
- Unir archivos.
- Filtrar registros.
- Crear columnas.

Al día siguiente vuelves a repetir exactamente el mismo proceso.

Y al siguiente también.

Y al siguiente...

---

# 😓 ¿Cuál es el problema?

El problema no es Excel.

El problema es que estás haciendo trabajo repetitivo.

Cada vez que realizas el mismo proceso manualmente:

- Pierdes tiempo.
- Puedes cometer errores.
- Los resultados pueden ser diferentes.
- No existe trazabilidad del proceso.

---

# 💡 Aquí aparece Power Query

Power Query fue creado para automatizar todas esas transformaciones.

En lugar de repetir el trabajo cada día, simplemente le enseñas a Excel qué debe hacer una sola vez.

Después únicamente presionas:

**Actualizar**

y Power Query repite automáticamente todas las transformaciones.

---

# 📦 Un ejemplo sencillo

Sin Power Query:

```text
Abrir archivo

↓

Eliminar columnas

↓

Cambiar nombres

↓

Aplicar filtros

↓

Guardar

↓

Repetir mañana
```

Con Power Query:

```text
Abrir archivo

↓

Actualizar

↓

Listo ✅
```

---

# 🏢 Caso empresarial

En el desarrollo del **Generador de Informes ANS**, diariamente se reciben archivos provenientes de Fénix.

Estos archivos requieren:

- Limpieza.
- Conversión de fechas.
- Eliminación de columnas innecesarias.
- Validación de información.
- Consolidación.

Realizar estas tareas manualmente consumiría mucho tiempo.

Power Query permite automatizar completamente ese proceso.

---

# 💼 Buenas prácticas

✅ Utilizar Power Query para transformar datos.

✅ Mantener los datos originales sin modificar.

✅ Evitar realizar limpieza manual directamente sobre los archivos fuente.

---

# ⚠️ Errores frecuentes

❌ Copiar y pegar información todos los días.

❌ Modificar manualmente los archivos originales.

❌ Repetir procesos que pueden automatizarse.

---

# 📝 Resumen

En este módulo aprendimos que:

- Power Query automatiza procesos repetitivos.
- Su objetivo principal es transformar datos.
- Siempre debe ser el primer paso dentro de un proyecto de Excel BI.
